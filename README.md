# communicationforC-J
好，现在先进入**项目启动/要件定义第一步**。

你这个系统先不要定义成“翻译工具”，而要定义成：

> 帮助在日本教会生活中的华语圣徒，进行自然、合适、有分寸的中日交流。

第一步先钉死这 5 件事：

## 1. 系统暂定名称

可以先叫：

> 中日教会交流辅助系统

之后再取正式名字。

## 2. 目标用户

第一版先锁定：

> 在日本生活、日语还不够自然，但需要和日本弟兄姊妹交通的华语圣徒。

不要一开始做给所有人。

## 3. 核心痛点

用户不是完全不会日语，而是：

> 不知道属灵场景下，怎么说才自然、温和、得体、不生硬。

## 4. 第一版核心功能

第一版只做一个功能：

> 输入中文内容 → 选择场景/语气 → 生成自然日文回复。

例如场景：

* 聚会请假
* 感谢照顾
* 回复交通
* 分享感想
* 邀约聚会
* 福音接触
* 祷告交通

## 5. 暂时不做

第一版先不做：

* 登录注册
* 自动读取 LINE/微信
* 群聊管理
* 语音识别
* 圣经搜索
* 复杂社交功能

先把一个流程跑通：

> 中文输入 → AI处理 → 日文输出 → 一键复制

你现在下一步就做这个文件：

```text
product-definition.md
```

内容先写：

```markdown
# 中日教会交流辅助系统

## 一句话定义
帮助在日本教会生活中的华语圣徒，生成自然、温和、合适的日文交流内容。

## 目标用户
在日本生活、日语还不够自然、需要和日本弟兄姊妹交通的华语圣徒。

## 核心痛点
用户知道自己想表达什么，但不知道如何用自然、属灵、得体的日文表达。

## 第一版核心功能
输入中文内容，选择交流场景和语气，系统生成适合 LINE 或当面交通使用的日文表达。

## 第一版使用流程
1. 用户输入中文
2. 选择场景
3. 选择语气
4. 点击生成
5. 复制日文回复

## 第一版暂时不做
登录、自动读取聊天软件、语音、群聊管理、复杂数据库。
```

这就是你的**第一个正式项目文档**。
