# 中日召会交流辅助系统｜UI设计书（MVP Ver 0.1）

## 1. 设计目标

本页面用于帮助在日本召会生活中的华语圣徒，将想表达的中文内容生成自然、温和、合适的日文交流内容。

MVP 第一版只做一条完整流程：

> 输入中文 → 选择场景 → 选择语气 → 生成日文 → 复制发送

---

## 2. 页面名称

**中日召会交流辅助系统**

副标题：

> 帮助你生成自然、温和、合适的日文交流内容

---

## 3. 页面整体结构

本系统第一版采用**单页面设计**，避免复杂跳转。

页面主要分为 5 个区域：

1. 顶部标题区
2. 中文输入区
3. 场景与语气选择区
4. 生成操作区
5. 结果展示与复制区

---

## 4. 用户使用流程

1. 用户打开页面
2. 输入想表达的中文内容
3. 选择交流场景
4. 选择语气风格
5. 点击「生成日文」
6. 系统显示生成中的 loading 状态
7. AI 返回日文结果
8. 用户点击「复制」
9. 用户将结果粘贴到 LINE、微信或其他聊天工具中发送

---

## 5. 页面元素设计

### 5.1 顶部标题区

| 项目 | 内容 |
|---|---|
| 标题 | 中日召会交流辅助系统 |
| 副标题 | 帮助你生成自然、温和、合适的日文交流内容 |
| 图标 | 使用普通交流/聊天图标，不使用十字架 |

---

### 5.2 中文输入区

| 项目 | 内容 |
|---|---|
| 区域标题 | 输入中文内容 |
| 输入形式 | 多行文本框 textarea |
| 字数限制 | MVP 建议 300 字以内 |
| placeholder | 例如：谢谢弟兄关心 / 今天可能没办法参加聚会 / 很受提醒 / 一起祷告 |
| 辅助显示 | 右下角显示字数：0 / 300 |

输入框要求：

- 支持换行
- 内容为空时不能生成
- 超过字数时提示用户缩短内容

---

### 5.3 场景选择区

| 项目 | 内容 |
|---|---|
| 区域标题 | 场景选择 |
| 表现形式 | 按钮组或下拉框 |
| 默认值 | 日常交流 |

第一版场景选项：

1. 日常交流
2. 聚会交通
3. 感谢
4. 请假
5. 福音接触
6. 祷告交通
7. 其他

说明：

> 场景用于帮助 AI 理解用户当前交流背景，生成更贴合实际场合的日文表达。

---

### 5.4 语气选择区

| 项目 | 内容 |
|---|---|
| 区域标题 | 语气选择 |
| 表现形式 | 单选按钮 radio |
| 默认值 | 简单自然 |

第一版语气选项：

1. 简单自然
2. 温和
3. 正式
4. 属灵交通

说明：

> 语气用于控制日文表达的礼貌程度、属灵表达浓度和自然程度。

---

### 5.5 生成操作区

| 项目 | 内容 |
|---|---|
| 按钮文字 | 生成日文 |
| 按钮类型 | 主按钮 |
| 点击条件 | 中文输入不为空 |
| 点击后状态 | 按钮禁用，显示 loading |

按钮状态：

#### 默认状态

显示：

> 生成日文

#### 生成中状态

显示：

> 正在生成...

同时按钮 disabled，防止重复点击。

#### 失败状态

显示错误提示：

> 生成失败，请稍后再试。

---

### 5.6 结果展示区

| 项目 | 内容 |
|---|---|
| 区域标题 | 生成结果 |
| 展示内容 | AI 生成的日文表达 |
| 结果为空时 | 显示 placeholder |
| 示例 placeholder | 这里将显示生成的日文内容... |

结果区要求：

- 支持多行文本
- 保留换行
- 生成结果清楚可复制
- 结果区域不要太小，方便用户检查内容

---

### 5.7 复制操作区

| 项目 | 内容 |
|---|---|
| 按钮文字 | 复制 |
| 点击后动作 | 将生成结果复制到剪贴板 |
| 成功提示 | 已复制！ |

复制成功后显示：

> 已复制！

2 秒后恢复为：

> 复制

如果没有生成结果，则复制按钮不可用或提示：

> 请先生成日文内容。

---

## 6. 页面状态设计

### 6.1 初始状态

- 输入框为空
- 场景默认：日常交流
- 语气默认：简单自然
- 生成按钮可见
- 结果区为空

### 6.2 输入中状态

- 用户输入中文
- 字数实时变化
- 输入不为空后可以点击生成

### 6.3 生成中状态

- 生成按钮禁用
- 显示「正在生成...」
- 可显示 loading 图标
- 防止重复请求

### 6.4 生成成功状态

- 结果区显示日文内容
- 复制按钮可点击

### 6.5 复制成功状态

- 显示「已复制！」
- 2 秒后恢复按钮状态

### 6.6 生成失败状态

- 显示错误提示
- 按钮恢复可点击
- 用户可以重新生成

## 7. MVP API 设计

### 7.1 接口名称

`POST /api/generate`

### 7.2 Request

```json
{
  "message": "谢谢弟兄关心",
  "scene": "感谢",
  "tone": "温和"
}
```

### 7.3 Response

```json
{
  "success": true,
  "result": "お気遣いありがとうございます。今日の交わりにも感謝します。とても励まされました。"
}
```

### 7.4 Error Response

```json
{
  "success": false,
  "error": "生成失败，请稍后再试。"
}
```

## 10. 第一版不做的功能

为了确保 MVP 快速完成，第一版暂时不做：

- 登录注册
- 用户历史记录
- 自动读取 LINE / 微信
- 群聊自动回复
- 语音输入
- 图片识别
- 多语言切换
- 复杂数据库
- AI长期记忆
- 手机 App

---

## 11. 验收标准

第一版完成时，需要满足以下条件：

1. 页面能正常打开
2. 用户可以输入中文
3. 用户可以选择场景
4. 用户可以选择语气
5. 点击「生成日文」后能调用后端 API
6. 能显示生成中的 loading 状态
7. 能显示 AI 返回的日文结果
8. 能一键复制生成结果
9. 输入为空时不能生成
10. API 出错时有错误提示
